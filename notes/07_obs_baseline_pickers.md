# Picker comparison on day-26 (2019-12-26)

Goal: before scaffolding fine-tuning, run every readily-available picker variant on a single verification day and see which combination is the most promising operating point for the year run.

**Ground truth:** mag07 trusted subset of the manual catalog. On 2019-12-26 this is **23 picks across 2 events**:
- `nllmaleen_mag07_202210_e1012` — origin 13:43:42.700 UTC, **6 P + 7 S** picks on **7 stations**
- `nllmaleen_mag07_202210_e1013` — origin 23:51:10.620 UTC, **5 P + 5 S** picks on **6 stations**

**Validation tolerances:** P ±0.5 s, S ±1.0 s (script defaults in `05_validate_picks.py`).

## Key observations

### 1. P-vs-S balance differs dramatically by training data
- Land-trained `instance` models (rows 1, 2): **P/S ≈ 5** — very P-biased.
- PickBlue (OBS-tuned): P/S **0.65–1.67** — closer to the catalog (0.82). PickBlue EQT @ 0.1 is the closest single-model match.
- OBSTransformer: P/S **0.25–0.53** — strongly S-biased. Reflects OBS-data characteristics (strong S on horizontals from converted/water-borne phases).

### 2. PhaseNet `instance` is the best P picker on this day (0.636)
…even outperforming all three OBS-specialized variants. *Caveat:* 11 manual P picks → flipping 2 changes recall by ~18 points. The year run will tell us if this is signal or noise.

### 3. OBSTransformer's threshold robustness is real (and unique)
S recall stays at **1.00 from 0.1 to 0.5** for OBSTransformer; only breaks at 0.7 (loses 1 of 12).
PickBlue PN and EQT lose substantial recall at 0.3:
- PickBlue PN @ 0.3: P-rec 0.55 → 0.27 (lost 3 P TPs), S-rec 0.58 → 0.42
- PickBlue EQT @ 0.3: P-rec 0.45 → 0.27, S-rec 0.92 → 0.75

→ **OBSTransformer can be tuned conservatively (thresh 0.5) without recall loss.** PickBlue must be run permissively (thresh 0.1) to preserve recall, paying in pick volume.

### 4. OBSTransformer @ 0.5 is the recommended OBS S-picker operating point
Strict win vs OBSTransformer @ 0.1: same perfect S recall, **3× fewer S FPs** (12,334 → 4,198), **70% reduction in total picks** (70,480 → 20,919). Confirms Niksejel & Zhang's threshold-robustness claim.

### 5. Recommended hybrid for the year run
**PhaseNet `instance` (P-thresh 0.1) for P + OBSTransformer `obst2024` (S-thresh 0.5) for S.**
- Best P recall on this day (0.636) + perfect S recall (1.00)
- Combined volume: ~5,800 P + ~15,700 S = ~21,500 picks/day — vs OBSTransformer @ 0.1 alone (70,480/day) → ~70% reduction
- Hands PyOcto a clean catalog with both phases at their best operating points
- **Caveat:** all of this is from one day with only 23 manual picks. Re-run on a few sampled days from the year before locking in.

## Implications for the fine-tuning plan
- **Don't fine-tune yet.** Off-the-shelf OBS weights (especially OBSTransformer 2024) already capture the OBS-noise adaptation we'd be retraining for, with a generous safety margin on threshold.
- The biggest remaining gap is **P recall** (best so far is PhaseNet `instance` @ 0.636). If fine-tuning is worth doing, target this — but only after the year-run statistics are real.
- See [`06_finetuning_plan.md`](06_finetuning_plan.md) for the deferred design.
