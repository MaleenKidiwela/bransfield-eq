## v1 discriminator — temporal-only (`scripts/discriminate_shots.py`)

Logic: for each pyocto event, find the nearest shot in time via `pd.merge_asof(..., direction="nearest", tolerance="N s")`. If within ±tol-sec → flag as shot.

### Why ±5 s wasn't enough

Two physical causes that user pinpointed:

1. **Pyocto origin-time error grows with shot-to-array distance.** Orca surveys are in the middle of the OBS array (sub-second origin-time fits); Phase 2 surveys (Rift, EDA, Other) are 20-50 km offshore (origin-time error can be 10-30 s).
2. **Water-column multiples and P/S converted phases.** A single airgun pulse produces P, water-multiples, Moho refractions, and basement P→S conversions, each arriving as separate coherent pulses up to ~30 s after the shot. PhaseNet picks them; pyocto associates them as "events."

## v2 discriminator — semi-supervised spectral classifier

User's insight: shots have a distinct narrow-band spectrum (airgun bubble pulses) vs broadband earthquakes. Plan:

1. **Extract per-event spectra** (`scripts/extract_event_spectra.py`):
   - Reuse `growclust/picker_only/pick_windows.npy` (483 k × 300 float32, ±1.5 s @ 100 Hz).
   - Per pick: `rfft → |·|² → log1p → unit-area normalize` → 151-D log power spectrum.
   - Per event: average across all valid picks.
   - Output: `catalogs/event_spectra.npy` (42,040 × 151 float32) + meta parquet.

2. **Train classifier** (`scripts/train_shot_classifier.py`):
   - **Labels (semi-supervised):**
     - SHOT = ±5 s temporal-flagged events in Jan 21 – Feb 4 (~8,900 events).
     - EQ = events outside Jan 21 – Feb 4 (~32,000 events). Pure because shotfiles cover exactly that window.
     - UNLABELED = unflagged events in Jan 21 – Feb 4 (gray zone).
   - **Model:** sklearn `HistGradientBoostingClassifier` on the 151-D spectra, `class_weight="balanced"`, 200 iter.
   - **Held-out AUC: 0.9985.** 98 % precision/recall on SHOT class. Confusion matrix on 7,872 held-out events: TN=6,040, FP=36, FN=32, TP=1,764.
   - **In-window classifier verdict at p ≥ 0.5** added **1,542 more events** beyond the temporal flag — these are the water-multiples and P/S conversions user predicted.
   - Out-of-window events default to `flag_shot_v2 = False` regardless of classifier output (avoids edge-effect false positives).

3. **Mean-spectrum diagnostic** (`notes/figures/event_spectra_v2_means.png`):
   - **EQ mean**: broadband, peak 0-2 Hz, smooth decay.
   - **SHOT mean**: narrowband 5-7 Hz peak (airgun source), plus 11-12 Hz and 15-17 Hz bumps (water-column bubble reverberations). Clearly distinct shape.

4. **Combined flag** (`flag_shot_v2`): True if (temporal ±5 s flag) OR (in-window AND classifier prob ≥ 0.5). **Total v2 flagged: 10,524 events (25 % of catalog).**

5. **Apply v2** (`scripts/apply_v2_clean.py`): wrote `pyocto_events_picker_only_no_shots_v2.csv` (31,516 events) + jan2019 subset (1,999 events). Also overwrote the canonical `_no_shots.csv` filenames so downstream pipeline picks up v2.

## Critical assessment: why Stage B-jan2019 looked "best" originally

User raised: was Stage B-jan2019's apparent cleanliness real or shot-biased?

Honest answer: **partially shot-biased.** The original Jan 2019 hypoDD catalog (5,610 events) was ~74 % shot events. K-means centroids on that point cloud would have been pulled along the survey tracks. Sub-region partitioning thus had a geometric imprint of the survey grids — and that imprint propagated into hypoDD's relative-position estimates for the real earthquakes inside each sub-region.

The v2-cleaned Stage B-jan2019 (1,999-event clean backbone, 7,624 final events) is what removes this bias. To validate, we're running v2-clean **Stage B** (full-year backbone, 271 events from cleaned strict prune) — if it agrees with v2-clean Stage B-jan2019 on swarm centroid + depth band, the cleaning was sufficient.

## Key lessons

1. **Active-source experiments contaminate ML pickers + associators heavily.** PhaseNet + pyocto can't distinguish airgun arrivals from real P phases; without filtering, 25 % of the Bransfield catalog was shot-derived.
2. **Pure temporal discrimination is insufficient for distant surveys.** Phase 2 needed spectral signature.
3. **Semi-supervised labels can come from data structure.** Shots only fire in a known window → that's a free label generator. No manual labeling needed.
4. **HGBT on raw 151-D spectra hit AUC 0.998.** No autoencoder needed — the spectra are linearly separable enough that gradient-boosted trees nail it.
5. **K-means on a contaminated catalog inherits the contamination geometry.** Original Stage B-jan2019's apparent cleanliness was partly shot-track imprint, not real swarm structure.

### Additional lesson

**Manual-anchored sub-clustering is feasible even when manual picks have no event locations.** ISTART=1 + array-centroid trial source lets hypoDD bootstrap absolute locations from the differential times alone. The 1,743-event manual backbone partitions cleanly into 16 K-means regions that successfully anchor the ML catalog. The one failure (subma_4) was driven by sparse anchor density (10 backbone events) plus depth clamping — future work might warm-start with deeper trial depth or drop sub-regions with <20 backbone seeds.
